<h1>SSH Flags </h1>
To connect a server using SSH, you will need to connect and log into the specified destination, which may be specified as either user@server-address or the domain.
 ```console 
 $ # real world example to ssh into remote server
 $ ssh vagrant@192.168.1.10 
 ```

 ```console
 ssh ubuntu@192.168.1.10
 or
 ssh user@example.com
 ```

<h3>Flags</h3>
Example of SSH to a remote server using Saturday's classes
 ```console
 $ ssh vagrant@192.168.1.10 -i ~/.ssh/id_ed25519 -p 2222
 ```
 We already know how to connect to a server using ssh and the destination
 ```console
 $ ssh vagrant@192.168.1.10
 ```
 ```console
 $ # used to select which private key file to use
 $ # -i lets me choose which identify to authenticate with instead of using my default key
 $ # before doing that the server must already have the matching public key.
 $ ssh prosper@192.168.1.10 -i ~/.ssh/id_ed25519
 $ # ssh to prosper server and using my private key in .ssh/id_ed25519

 $ # port to connect to on the remote host
 $ # by default, SSH connects to port 22. 
 $ # if the server is running SSH on a different port, use -p
 $ ssh -p 2222 prosper@192.168.1.10
 $ # connects to prosper's server on port 2222, instead of default 22

 $ # makes the SSH porint detailed information about what's happening during the connection process
 $ # which config files it's reading
 $ # which keys it's trying
 $ # the handshake/ authentication steps
 $ # where it's connecting to and on what port
 $ # -v stands for verbose and mainly used for debugging. If your SSH connection is failing or hanging use -v.
 $ -v -basic verbose
 $ -vv -more detail
 $ -vvv maximum detail
 $ ssh -v prosper@192.168.1.10
 
 $ # forces ssh to use IPv4 address only 
 ssh -4 user@example.com

 $ forces ssh to use IPv6 address only 
 ssh -6 user@example.com
 ```
