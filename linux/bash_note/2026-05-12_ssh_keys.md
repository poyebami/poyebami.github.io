<h1>SSH Keys </h1>
In SSH keys, there are two files. The `private` and `public` keys. 

Your private key is the key, your public key is the lock.
We use SSH to make a connection to a server. SSH reads the private key and tries to connect to the server using the public key. Once the server gets the public key, it creates a random string of characters and uses a algorithm to encrypt the string using the public key. It send the string to your computer and tells it to decrypt that string using the private key. Your computer sends the calculation back to the serve and that proves that you are the only trying to connect to the server. 

<h3>Private Key</h3>
The private key is meant ONLY for you and no one else. If someone get access to your private  key, they can pretend to be you when connecting to another computer or service. They could access servers, repositories, or cloud sysyem. Private key's bypasses password in many setups. 

<h3>Public Key</h3>
It tells the server or service that anyone with the matching private key can log in. You give the public key to a server and your computer keys the private key. During login, the server checks in your private key mataches the stored public key.
